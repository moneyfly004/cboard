# MoneyFly UI Redesign V1

本设计稿面向 MoneyFly 当前 Flutter 客户端，覆盖 Android、Windows、macOS 三端。内容只描述客户端界面、交互和视觉规范，不包含后台结构、接口路径、服务部署、鉴权或内部数据细节。

## 设计目标

- 建立统一的 MoneyFly 产品视觉：专业、稳定、轻量，强调连接状态、账号订阅和设备使用。
- 三端共用同一套设计语言：移动端强调单手操作，桌面端强调信息密度和侧边导航，macOS 保持更轻的层级和原生留白。
- 所有主页面都使用一致的页面标题、卡片、列表、按钮、空状态、加载态和错误态。
- 关键操作保持明确：连接/断开、同步订阅、模式切换、节点测速、设备管理、订单支付、日志筛选。

### 组件规范

- 页面最大内容宽度：手机全宽，平板 `760dp`，桌面主内容 `1120dp`。
- 卡片圆角：`8dp`，小信息块圆角 `10dp`，按钮圆角采用 Material 3 默认圆角。
- 间距：使用 `4/8dp` 节奏；页面外边距手机 `16dp`，桌面 `24-32dp`。
- 触控：移动端所有可点区域不小于 `48dp`，桌面图标按钮不小于 `40dp`。
- 字体：继续使用当前本地化字体链，标题用 `titleMedium/titleLarge`，数据值用 `labelLarge/titleMedium`。
- 图标：继续使用 Material / Fluent 图标，不使用 emoji 做结构图标。

## 三端布局

### Android

- 底部导航保留 3 个主入口：主页、账户、设置。
- 首页从上到下为：顶部品牌栏、账户订阅卡、当前配置卡、连接按钮、当前节点卡、模式切换按钮。
- 模式切换固定在底部导航上方，底部安全距离不少于 `16dp`。
- 列表页面使用单列列表，FAB 用于同步、测速等主要动作。
- 长设置项进入二级页面，不在首页堆叠过多开关。

### Windows

- 左侧 NavigationRail 作为主导航，桌面宽度下展开文字。
- 主内容使用两栏或三栏密度：状态/订阅信息在顶部，列表和详情并列显示。
- 主页建议结构：左侧连接控制与订阅状态，右侧当前节点、速度统计、快速操作。
- 日志页面保留固定筛选栏，日志内容使用等宽感更强的紧凑列表。

### macOS

- 与 Windows 共用布局骨架，但视觉更轻：减少阴影，更多使用边框和留白。
- macOS 顶部栏保持简洁，主操作放在页面内部，避免 toolbar 过载。
- 弹层、菜单和刷新操作优先使用平台默认 Material/Cupertino 质感。

## 页面设计清单

| 页面 | 核心信息 | 优化方向 |
| --- | --- | --- |
| 主页 | 连接状态、订阅、配置、当前节点、模式切换 | 建立清晰层级，连接按钮居中，状态卡 2x2 固定布局 |
| 账户概览 | 用户状态、套餐、设备、快捷入口 | 先展示订阅健康度，再展示套餐/设备/订单入口 |
| 登录/注册/找回密码 | 表单输入、验证码、提交反馈 | 使用分段模式切换，表单字段保持 48dp 高度，错误反馈贴近字段 |
| 套餐购买 | 套餐、价格、支付入口 | 卡片化套餐，主行动按钮统一靠右/底部 |
| 订阅同步 | 当前订阅、导入状态、同步动作 | 强调可导入状态和最后同步结果 |
| 设备管理 | 在线状态、设备类型、备注、移除 | 在线设备优先，危险操作二次确认 |
| 订单记录 | 订单号、状态、金额、支付 | 状态标签颜色语义化，支持刷新状态 |
| 节点列表 | 选中节点、延迟、排序、测速 | 桌面网格、移动单列；选中态使用左侧色条和背景 |
| 配置列表 | 订阅配置、更新时间、流量 | 使用一致的 ProfileTile；主配置突出 active 状态 |
| 设置 | 常规、链式、路由、DNS、入站、TLS | 分组列表卡片化，二级页面使用统一页面宽度 |
| 路由规则 | 规则条件、动作、排序 | 条件 chips 保持可读，不让文本溢出 |
| 分应用代理 | 应用列表、代理模式、搜索 | 顶部模式说明，列表项保持图标、名称、包名三层级 |
| 日志 | 过滤、级别、暂停/清空、分享 | 筛选栏固定，日志行用低噪声分割线 |
| 关于 | 版本、许可、诊断入口 | 保持简洁，突出版本与更新状态 |

## 交互要求

- 连接按钮、同步、测速、支付、删除等操作必须有加载态或禁用态。
- 旧客户端检测到新版本时继续使用更新弹窗，不在页面中暴露内部发布机制。
- 启动/重启后应自动刷新账号订阅状态，并对可导入订阅重新生成本地配置。
- 如果用户关闭某个普通远程配置的自动更新，周期更新尊重关闭状态；启动时仍可执行一次恢复性刷新，避免重启后使用旧配置。
- 所有页面在 `375dp` 宽度下不得横向滚动，文本过长时换行或省略。
